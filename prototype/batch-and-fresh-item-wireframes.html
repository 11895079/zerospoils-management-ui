Shopping Batch and Fresh Item Vision

This wireframe breaks the capture model into two clean product paths. The FAB should only offer Add Item and Add Shopping Batch. Receipt photos and goods photos both live inside the shopping batch workflow, while single fresh-item vision stays inside the add-item workflow.

FAB Rule No separate top-level batch-photo entry. It belongs under shopping batch.
Batch Rule A shopping batch must save without OCR. Photos and OCR are assists, not prerequisites.
Fresh Item Rule Produce, meat, and cooked-food vision is a single-item flow, not part of receipt capture.

Screen Set

  • Inventory screen with corrected FAB menu
  • Shopping batch workspace with metadata, receipt photos, and optional goods-photo capture
  • Batch review screen that merges receipt and goods suggestions for confirmation
  • Batch history/detail as the persistent home of shopping trips
  • Add item screen with a separate fresh-item camera assist

1. Inventory Changes

  • The header receipt button can remain as a quick shortcut into shopping batches.
  • The FAB should not expose a separate batch-photo capture item.
  • Shopping batch is the parent workflow. Receipt photos and goods photos are capture tools inside it.
This is the key correction to the earlier layout: batch-photo capture is nested inside shopping batch, not shown as a parallel top-level mode.

2. Shopping Batch Workspace

  • Receipt photo capture and goods-photo capture are siblings inside the batch workspace.
  • Receipt photos support pricing and optional OCR of line items.
  • Goods photos support visual item suggestions to help map abbreviated receipt lines.
  • The batch still saves even when neither capture mode is used.
If you want to reduce scope, this screen can ship with receipt photos enabled first and goods-photo tiles present but marked “Coming next”.

3. Batch Review Logic

  • Receipt OCR is not the source of truth. It is just one source of candidates.
  • Goods photos help disambiguate shortened receipt names like “BNS”, “CHK THGHS”, or “ORG MLK”.
  • Anything uncertain is editable before save.
This is the screen where the shopping batch concept becomes valuable: it can combine pricing, image suggestions, and manual confirmation without pretending any one input is perfectly reliable.

5. Fresh Item Vision

  • This is the right home for fruit capture and non-barcoded food recognition.
  • It is intentionally separate from the shopping batch flow.
  • Fresh-item CV should suggest a shortlist and field hints, never silently save.
If you want a single sentence to govern the architecture: shopping batch is about a trip, fresh-item vision is about a single visible item.
Amendment points: You can use this file to decide whether goods-photo capture should ship immediately inside shopping batch or appear as a disabled/flagged sub-section first. The core structural choice shown here is that it is part of the shopping batch workflow, while fresh produce/meat/cooked-food recognition belongs on the add-item screen.
