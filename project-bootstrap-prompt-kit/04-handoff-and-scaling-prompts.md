# 04 - Handoff And Scaling Prompts

Use these prompts for onboarding, role transitions, and scaling execution.

## 1. Team Handoff Prompt

```text
Act as a Technical Delivery Manager preparing a clean handoff.

Inputs:
- Current project state
- Code/repo status
- Open risks and pending decisions

Produce:
1) Handoff brief (current status, what's done, what's pending)
2) Decision log and rationale summary
3) Top operational and technical risks
4) Next 2-week execution plan for incoming team
```

## 2. Executive Update Prompt

```text
Act as a Head of Engineering reporting to executives.

Inputs:
- Milestone status
- KPI trends
- Risks and mitigation actions

Create:
1) Concise executive summary
2) Progress against goals
3) Key risks and confidence level
4) Decisions needed from leadership
```

## 3. Scaling Prompt (1 team -> many teams)

```text
Act as a Platform and Delivery Scaling Advisor.

Inputs:
- Current architecture and delivery model
- New team count and expected scope growth

Provide:
1) Team topology and ownership model
2) Interface contracts and integration governance
3) Common standards (coding, testing, release, observability)
4) Cross-team dependency protocol
5) Scaling risk register and prevention controls
```
