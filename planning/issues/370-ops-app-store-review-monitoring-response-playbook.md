## Context
App store reviews are a primary feedback channel; responses affect conversion and trust.

## Goal
Create a review monitoring and response playbook.

## Expected behavior
- Team checks reviews on a defined cadence
- Responses are consistent with brand voice

## Acceptance criteria (Definition of Done)
- [ ] Create `docs/ops/reviews.md` with cadence, templates, and escalation triggers
- [ ] Define categories: feature request, bug, UX confusion, praise
- [ ] Add a process to convert reviews into GitHub issues
- [ ] Unit/widget/integration tests added or updated
- [ ] Telemetry added/updated (event names + key properties)
- [ ] Offline-first behavior verified (where applicable)
- [ ] Accessibility basics (labels, contrast, tap targets)

## Out of scope
- Automated review ingestion.
