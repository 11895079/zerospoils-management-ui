## Context
Frequent small releases reduce risk and help iterate fast.

## Goal
Define release cadence and changelog standards.

## Expected behavior
- Team can ship hotfixes quickly
- Users see clear changelogs

## Acceptance criteria (Definition of Done)
- [ ] Create `docs/ops/releases.md` covering: hotfix vs minor vs major
- [ ] Define changelog format and where it lives
- [ ] Add a lightweight 'release captain' checklist
- [ ] Unit/widget/integration tests added or updated
- [ ] Telemetry added/updated (event names + key properties)
- [ ] Offline-first behavior verified (where applicable)
- [ ] Accessibility basics (labels, contrast, tap targets)
