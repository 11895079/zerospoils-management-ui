# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in ZeroSpoils, please report it responsibly:

1. **Do NOT** open a public GitHub issue
2. Email security concerns to: [security contact - to be determined]
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Response Timeline
- Initial response: within 48 hours
- Security fix: as soon as possible
- Public disclosure: coordinated with reporter

## Security Practices

ZeroSpoils follows these security principles:
- No PII in telemetry by default
- Opt-in/opt-out for data collection
- Secrets never committed (use environment variables)
- Dependency scanning enabled
- Regular security audits planned post-launch

See `planning/docs/security-baseline.md` for detailed security architecture.
