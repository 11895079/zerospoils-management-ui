## Context
Receipt capture must feel effortless and privacy-respecting.

## Goal
Implement receipt capture UI flow and storage for processing.

## Expected behavior
- User can photograph a receipt or import an image
- User sees what data is extracted and can approve

## Acceptance criteria (Definition of Done)
- [ ] Receipt capture screen implemented

- [ ] Unit/widget/integration tests added or updated
- [ ] Telemetry added/updated (event names + key properties)
- [ ] Offline-first behavior verified (where applicable)
- [ ] Accessibility basics (labels, contrast, tap targets)
